BTS SIO — SISR — Fiche Situation Professionnelle

Active Directory Samba4 sous Linux
GPO + Automatisation comptes + Portail reset

📅 Projet personnel BTS SIO 🏢 Environnement Proxmox (labo) 💻 Debian 12 + Windows 11
Contexte

Windows Server peut etre couteux, demande une maintenance reguliere et reste une cible frequente des attaques. L'objectif de ce projet est de demontrer qu'il est possible de deployer un Active Directory complet et compatible Windows en utilisant uniquement Samba4 sur Linux (Debian 12), sans aucune licence propriataire. Le projet couvre l'ensemble du cycle : installation du DC, gestion des GPO, automatisation des comptes eleves, delegation de droits aux professeurs et portail de reinitialisation.

Objectifs
  • Installer et configurer un controleur de domaine Samba4 sur Debian avec DNS interne (SRV LDAP + Kerberos)
  • Creer une structure AD : OU Eleves (SIO1, SIO2), OU Enseignants, OU Groupes
  • Joindre un poste Windows au domaine BTS-SIO.LOCAL
  • Mettre en place une GPO eleves : fond d'ecran impose + restrictions (CMD, Regedit, Task Manager, Panneau de config)
  • Automatiser la creation de 60 comptes (30 SIO1 + 30 SIO2) avec mot de passe provisoire et changement obligatoire a la 1ere connexion
  • Preparer un script de reinitialisation annuelle + export CSV pour distribution aux eleves
  • Permettre aux professeurs de reinitialiser les mots de passe via delegation AD et portail web (LAM)
Realisations
1 Configuration reseau et installation Samba4
  • IP fixe sur Debian via NetworkManager : 192.168.26.10/24, gateway 192.168.26.254
  • Synchronisation heure avec chrony (Kerberos echoue si decalage > 5 min)
  • Installation : samba smbclient winbind krb5-user dnsutils
  • Desactivation des services legacy (smbd nmbd winbind) pour eviter les conflits
  • Provisionnement : samba-tool domain provision --use-rfc2307 --interactive
  • DNS forwarder vers 8.8.8.8 pour la resolution internet
2 Verification DNS (point critique AD)
  • Le DNS est le coeur de l'AD : sans enregistrements SRV corrects, rien ne fonctionne
  • Verification SRV LDAP (port 389) et Kerberos (port 88) depuis le DC : host -t SRV _ldap._tcp.bts-sio.local 127.0.0.1
  • Verification depuis Windows : nslookup -type=SRV _ldap._tcp.bts-sio.local 192.168.26.10
  • Verification partages SYSVOL et NETLOGON : smbclient -L localhost -N
3 Structure Active Directory
  • Creation des OU avec samba-tool ou create : Eleves, SIO1, SIO2, Enseignants, Groupes
  • Verification visuelle dans dsa.msc (RSAT installe sur WIN-ADMIN)
  • Installation RSAT : Add-WindowsCapability -Online -Name Rsat.ActiveDirectory.DS-LDS.Tools~~~~0.0.1.0
4 Jointure domaine Windows
  • Prerequis absolu : DNS du poste Windows = 192.168.26.10 (sans ca, jointure impossible)
  • Jointure via sysdm.cpl → domaine bts-sio.local → identifiants BTS-SIO\Administrator
  • Verification : nslookup bts-sio.local 192.168.26.10 depuis Windows
5 GPO eleves — GPO_ELEVES_RESTRICTION
  • Creation et liaison dans gpmc.msc sur OU=SIO1 et OU=SIO2
  • Fond d'ecran impose : chemin UNC \\bts-sio.local\sysvol\bts-sio.local\scripts\fond_ecran\fond.jpg
  • Restrictions : Panneau de config, Gestionnaire des taches, Regedit, CMD — tous bloques
  • Filtrage securite : Authenticated Users doit rester present (sinon GPO refusee)
  • Verification : gpupdate /force puis gpresult /r — GPO_ELEVES_RESTRICTION doit apparaitre dans "appliquees"
  • Preuve : gpresult /h C:\gpresult.html
6 Automatisation des 60 comptes eleves (script bash)
  • Script create_eleves_sio.sh : idempotent (cree ou met a jour si le compte existe deja)
  • 30 comptes SIO1 : sio1-01 a sio1-30 dans OU=SIO1,OU=Eleves
  • 30 comptes SIO2 : sio2-01 a sio2-30 dans OU=SIO2,OU=Eleves
  • Mot de passe provisoire : Eleve@2025 + flag --must-change-at-next-login
  • Compte active explicitement apres creation (evite "compte desactive" au login)
  • Verification : samba-tool user list | egrep '^sio[12]-'
7 Reset annuel + export CSV
  • Script reset_eleves_passwords.sh : reinitialise tous les comptes sio[12]-* en une commande
  • Script export_eleves_csv.sh : genere un CSV login;classe;motdepasse_provisoire pour distribution
8 Delegation professeurs (principe du moindre privilege)
  • Groupes Prof_ResetPwd_SIO1 et Prof_ResetPwd_SIO2 crees dans l'AD
  • Delegation dans dsa.msc sur OU=SIO1/SIO2 : reset password uniquement, sans droits admin
  • Option portail LAM (LDAP Account Manager) : apache2 + php-ldap + ldap-account-manager
  • Le prof se connecte au portail et ses droits sont limites par la delegation
Points techniques cles a retenir
DNS = coeur de l'AD Sans SRV LDAP (port 389) et Kerberos (port 88) corrects, le domaine ne fonctionne pas. Toujours verifier avec host -t SRV depuis le DC.
Kerberos et l'heure Si le decalage horaire entre machines depasse 5 minutes, Kerberos refuse l'authentification. Solution : chrony sur le DC.
GPO_ELEVES_RESTRICTION Le filtrage securite doit conserver Authenticated Users. Si ce groupe est supprime, la GPO est systematiquement refusee.
samba-tool gpo setlink Ordre important : DN de l'OU en premier, nom de la GPO en second. Sinon erreur "does not exist".
Difficultes rencontrees et solutions
Probleme 1 — nslookup timeout / jointure domaine impossible
Cause : DNS du poste Windows non pointe sur 192.168.26.10.
Solution : ipconfig /all pour verifier, puis forcer DNS = IP du DC. Verifier aussi que samba-ad-dc ecoute sur le port 53 : sudo ss -tulpn | grep ':53'
Probleme 2 — GPO refusee (gpresult montre "Refusee")
Causes possibles : GPO pas liee a la bonne OU, utilisateur pas dans l'OU ciblee, Authenticated Users supprime du filtrage de securite, pas de gpupdate apres modification.
Solution : verifier dans dsa.msc et gpmc.msc, puis gpupdate /force.
Probleme 3 — Fond d'ecran non applique malgre GPO active
Cause : chemin local au lieu de chemin UNC.
Chemin correct : \\bts-sio.local\sysvol\bts-sio.local\scripts\fond_ecran\fond.jpg
Test : copier le chemin dans l'explorateur Windows — le fichier doit apparaitre.
Probleme 4 — Compte "desactive" au login
Diagnostic : samba-tool user show sio1-05 | grep -i userAccountControl
Solution : samba-tool user enable sio1-05

Commandes cles a maitriser pour l'oral
Verifier le DC :
sudo systemctl status samba-ad-dc --no-pager
Verifier DNS SRV :
host -t SRV _ldap._tcp.bts-sio.local 127.0.0.1
Lister les comptes eleves :
samba-tool user list | egrep '^sio[12]-'
Forcer GPO Windows :
gpupdate /force && gpresult /r
Activer un compte :
samba-tool user enable sio1-05
Reinitialiser mdp :
samba-tool user setpassword sio1-05 --newpassword="Eleve@2025" --must-change-at-next-login
